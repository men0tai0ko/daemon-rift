## ■ 次にやるべきこと（最重要）

### 優先度1: アイテム画面UIの実装
**何をするか:** 仲魔一覧画面（screen-party）内にアイテムタブを追加、または探索画面内にアイテムポーチを追加する。

**実装方針（推奨）:**
```
仲魔画面にタブを追加: [仲魔一覧] [アイテム]
アイテムタブの内容:
  - ITEM.list() で所持アイテム一覧取得
  - 各アイテムに「使う」ボタン
  - ITEM.use(name, target) を呼び出し
  - 使用後 renderPartyBar() / renderExplore() で更新
```

**呼び出すAPI:**
```javascript
ITEM.list()           // → [{name, qty, icon, desc}, ...]
ITEM.use('回復薬', STATE.party[0])  // → {ok, msg}
ITEM.count('スキル石') // → 数値
```

### 優先度2: 戦闘中アイテム使用
**何をするか:** 戦闘画面に「アイテム」ボタンを追加、回復薬・御札を戦闘中に使えるようにする。

### 優先度3: SAVE_SCHEMA_VERSION を 2 に更新
**何をするか:** `items` フィールド追加に伴うスキーマバージョンアップ。
```javascript
// index.html 837行付近

```
旧セーブデータは自動破棄される（`loadGame()` に実装済み）。

## ■ 注意点

### 触ると壊れやすい箇所

| 箇所 | 理由 |
|------|------|
| `SAVE_KEYS` 配列 | ここにフィールドを追加し忘れると saveGame() でセーブされない |
| `SAVE_SCHEMA_VERSION` | 変更すると既存セーブデータが全破棄される |
| `BATTLE._enemyAttack()` | `_guardActive` フラグのリセット忘れに注意 |
| `FUSION.execute()` | `_skillStoneActive` フラグのリセット忘れに注意 |
| `G.continueGame()` | `SAVE_KEYS.forEach` で STATE を復元するため、SAVE_KEYS に含まれないフィールドはリセットされない |
| `UI` オブジェクトのメソッド名 | `renderExplore` 等のグローバル後方互換ラッパーが並存しているため、同名のグローバル関数を新たに定義しないこと |

### 設計上の制約

- **1ファイル完結** が最重要制約。外部ファイル・CDN依存禁止（Web Audio APIのみ例外）
- **STATE はオブジェクト参照渡し** のため、party/storage の要素はどこからでも直接変更可能。意図しない副作用に注意
- バージョン番号はコメントヘッダー・metaタグ・`APP_VERSION` 定数の3箇所を必ず同期すること
